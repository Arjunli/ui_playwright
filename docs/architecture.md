# 架构设计文档（参考八爪鱼）

## 核心架构

本框架参考八爪鱼的设计理念，采用分层架构和核心引擎模式。

```
┌─────────────────────────────────────────────────────────┐
│                    应用层 (Application)                  │
│  - Recorder (录制器)                                     │
│  - Executor (执行器)                                     │
│  - Converter (转换器)                                    │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                    核心层 (Core)                         │
│  - LocatorEngine (定位引擎)                             │
│  - StrategyGenerator (策略生成器)                       │
│  - ElementFingerprintService (元素指纹服务)             │
│  - StabilityScoringService (稳定性评分服务)             │
│  - SmartRetry (智能重试)                                │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                    适配层 (Adapter)                      │
│  - LocatorResolver (定位解析适配器)                     │
│  - StepRunner (步骤执行适配器)                          │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                    基础层 (Foundation)                   │
│  - Playwright API                                       │
│  - Type Definitions                                     │
│  - Utilities                                            │
└─────────────────────────────────────────────────────────┘
```

## 核心模块说明

### 1. LocatorEngine (定位引擎)

**职责**：统一管理所有定位策略，提供智能定位和重试机制

**核心特性**：
- 多策略Fallback机制：依次尝试所有策略直到成功
- 智能重试：自动重试失败的策略
- 动态优先级调整：根据历史成功率调整策略优先级
- 页面状态检测：自动检测页面关闭等情况

**使用示例**：
```typescript
const engine = new LocatorEngine(page);
const locator = await engine.resolve(locatorConfig);
```

### 2. StrategyGenerator (策略生成器)

**职责**：基于元素信息生成多种定位策略

**核心特性**：
- 元素指纹识别：结合多个属性生成唯一标识
- 相对XPath生成：基于父元素、兄弟元素生成相对路径
- 稳定性评估：为每个策略计算稳定性分数
- 优先级排序：按稳定性自动排序策略

**策略优先级**：
1. testid (1) - 最稳定
2. id (2) - 静态ID
3. role (3) - 语义化属性
4. name (4) - 表单元素
5. placeholder (5) - 输入框
6. xpath (5.8) - 基于元素属性
7. text (6) - 文本内容
8. css (7) - CSS选择器

### 3. ElementFingerprintService (元素指纹服务)

**职责**：计算元素指纹，评估属性稳定性

**核心特性**：
- 多属性组合：tagName + id + className + text + position
- 稳定性评分：为每个属性计算稳定性分数
- 元素签名：生成唯一标识，即使部分属性变化也能识别

**稳定性分数**：
- testid: +10
- id (静态): +8
- name: +7
- placeholder: +6
- role: +5
- className (稳定): +4
- text: +3
- tagName: +1

### 4. StabilityScoringService (稳定性评分服务)

**职责**：评估定位策略的稳定性，动态调整优先级

**核心特性**：
- 历史成功率记录：记录每个策略的成功/失败次数
- 动态优先级调整：根据成功率自动调整优先级
- 策略排名：为所有策略计算稳定性排名

### 5. SmartRetry (智能重试)

**职责**：提供智能重试和容错机制

**核心特性**：
- 自动重试：失败时自动重试，支持指数退避
- 降级策略：主策略失败时自动尝试降级策略
- 多策略执行：依次尝试多个策略直到成功

## 设计原则

### 1. 单一职责原则
每个模块只负责一个明确的功能，便于维护和测试。

### 2. 开闭原则
对扩展开放，对修改关闭。新功能通过扩展而非修改实现。

### 3. 依赖倒置原则
高层模块不依赖低层模块，都依赖抽象接口。

### 4. 接口隔离原则
使用多个专门的接口，而不是一个总接口。

## 数据流

### 录制流程
```
用户操作 → ActionCapture → LocatorGenerator → StrategyGenerator → LocatorConfig
```

### 执行流程
```
TestConfig → ConfigExecutor → StepRunner → LocatorResolver → LocatorEngine → Locator
```

## 扩展点

### 1. 自定义定位策略
实现 `LocatorStrategy` 接口，注册到 `LocatorEngine`。

### 2. 自定义重试策略
实现 `RetryOptions` 接口，配置 `SmartRetry`。

### 3. 自定义稳定性评分
扩展 `StabilityScoringService`，实现自定义评分算法。

## 性能优化

1. **缓存机制**：缓存元素指纹和策略生成结果
2. **并行执行**：支持并行执行多个定位策略
3. **早期退出**：找到有效策略后立即返回
4. **统计优化**：定期清理旧统计数据

## 最佳实践

1. **优先使用元素属性**：testid > id > name > placeholder
2. **避免动态属性**：过滤动态ID和随机类名
3. **使用相对路径**：避免绝对XPath
4. **记录历史数据**：利用历史成功率优化策略
5. **智能重试**：配置合理的重试次数和延迟
